Technical Notes

Admin panel updated — Hero merged, Coach removed, Currency added

The admin sidebar no longer has a "Coach Intro" section (the coach data merged into the Hero section on the live page). Hero preview now shows the coach photo with gradient overlay. An emotionalHook textarea was added to the Hero editor. A KES/USD currency radio toggle was added to the Services editor. Admin save still preserves all fields including coachIntro data.

Pine accent colour

Pine (#4A5E35) is used throughout as the accent colour: section heading labels, "Explore Services" button, blog tag pills, hover states, Services contact links, currency badge. Replaces cinnamon as the primary accent. Hero pills use rgba(74,94,53,0.2) background.

Text size floor: 14px (text-sm)

Every text-xs, text-[10px], text-[11px], and text-[12px] was replaced with text-sm (14px) across 10 files. No text on the site is smaller than 14px.

Admin save preserves all CMS fields

The admin panel loads the full content.json into state (including fields it doesn't edit: pillars, elements, faq, credentials, newsletter). These pass through untouched on save. Confirmed: no data loss on admin save.

Uploaded images are gitignored

/public/uploads/ is in app/.gitignore. Blog cover images and the coach photo (if uploaded via admin) will not be committed. On deploy, these must be present on the server or use a CDN/object storage. For Vercel: use Vercel Blob or commit the uploads folder separately.

Current coach photo (Umulkheiri.jpg) and all illustrations are in /public/images/ which IS committed. Only /public/uploads/ (admin-uploaded files) is excluded.
Deployment: Vercel (not Netlify)

This is a Next.js app — deploy to Vercel for zero-config. Custom domain on Vercel free plan is supported. The content.json write API (/api/admin/save) requires a writable filesystem — Vercel's serverless functions are read-only at runtime. For production admin editing, migrate to a database (PlanetScale, Supabase) or use Vercel's Edge Config / KV store.

For the launch MVP, the admin save will work on any Node.js host with a persistent filesystem (Railway, Render, Fly.io). Vercel is fine if content updates are made via direct content.json edits in the repo and redeployed.
